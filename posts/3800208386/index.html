How to use free command in Linux

2024-03-27
linux
3.5k words
  • free

The free command in Linux and UNIX systems is used to display the current system’s memory usage. It provides detailed information about physical memory, swap space, and kernel buffers. Here is a detailed explanation of the output of the free command:

Basic Output

When you run the free command, it typically displays information in the following format:

The breakdown is as follows:

  • Total Memory (total): The system has a total of 15GB of physical memory.

  • Used Memory (used): Currently, 8.1GB of physical memory is being used.

  • Free Memory (free): There are 221MB of physical memory that is free and unused.

  • Shared Memory (shared): There is 1.2GB of memory marked as shared, typically used by multiple processes.

  • Buffers (buffers): The system is using 328MB of memory for buffers, which is usually for file system buffering.

  • Cache (cache): The system is using 6.7GB of memory for cache, primarily for storing file system and other data to speed up access.

  • Available Memory (available): The system has 5.7GB of memory that is available, which includes free and cache memory that the system can use when needed.

  • Swap Space (Swap): The system has 15GB of swap space configured, with 3.6GB currently in use and 12GB available.

  • Total (Total): Combining physical memory and swap space, there is a total of 31GB of memory and swap space.

  • Used Total (used): The system has used 11GB of memory and swap space.

  • Free Total (free): There is 12GB of memory and swap space that is free.

Physical Memory Space

The columns are as follows:

  • total: The total amount of memory in the system.
  • used: The amount of memory currently in use, including buffers and cache.
  • free: The amount of completely free memory, excluding swap space.
  • shared: The total amount of memory shared by multiple processes, which is used by both physical memory and swap space.
  • buffers: The amount of memory used by the kernel for buffers, used for file system buffering.
  • cache: The amount of memory used by the kernel for caching, used for file system caching.

Swap Space

  • total: The total amount of swap space.
  • used: The amount of swap space currently in use.
  • free: The amount of unused swap space.

Options

Display Unit Options

  • -b, –bytes: Display memory usage in bytes.
  • –kilo: Display memory usage in kilobytes (KB).
  • –mega: Display memory usage in megabytes (MB).
  • –giga: Display memory usage in gigabytes (GB).
  • –tera: Display memory usage in terabytes (TB).
  • –peta: Display memory usage in petabytes (PB).
  • -k, –kibi: Display memory usage in kibibytes (KiB), 1 KiB = 1024 bytes.
  • -m, –mebi: Display memory usage in mebibytes (MiB), 1 MiB = 1024 KiB.
  • -g, –gibi: Display memory usage in gibibytes (GiB), 1 GiB = 1024 MiB.
  • –tebi: Display memory usage in tebibytes (TiB), 1 TiB = 1024 GiB.
  • –pebi: Display memory usage in pebibytes (PiB), 1 PiB = 1024 TiB.

Display Format Options

  • -h, –human: Display memory usage in a human-readable format, such as “1.5G” instead of “1536M”.
  • –si: Use the International System of Units (powers of 1000) instead of the binary unit system (powers of 1024).

Other Options

  • -l, –lohi: Display detailed low and high memory statistics.
  • -t, –total: Display the total of RAM and swap space.
  • -s N, –seconds N: Repeat printing memory usage every N seconds.
  • -c N, –count N: Repeat printing memory usage N times, then exit.
  • -w, –wide: Wide format output, without limiting the width of each column.

Help and Version Information Options

  • –help: Display help information and exit.
  • -V, –version: Output version information and exit.
